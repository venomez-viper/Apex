# IEEE Preparation Checklist

This checklist summarizes the repository and paper cleanup against standard
IEEE-style conference-paper expectations. It is based on the IEEE Author Center
template guidance, the IEEE Editorial Style Manual for Authors, and the
IEEEtran conference template conventions.

References:

- IEEE Template Selector: <https://template-selector.ieee.org/>
- IEEE Author Center article templates:
  <https://journals.ieeeauthorcenter.ieee.org/create-your-ieee-article/use-authoring-tools-and-ieee-article-templates/>
- IEEE Editorial Style Manual for Authors:
  <https://journals.ieeeauthorcenter.ieee.org/wp-content/uploads/sites/7/IEEE-Editorial-Style-Manual-for-Authors.pdf>

## Paper Structure

| Check | Status |
| --- | --- |
| IEEEtran conference class used | Done |
| Title is specific and technical | Done |
| Author block uses author name only | Done |
| Abstract is a single compact summary | Done |
| IEEE keywords are present | Done |
| Sections are numbered through IEEEtran | Done |
| Figures and table are referenced in the paper | Done |
| Bibliography uses IEEEtran style | Done |
| Code and artifact availability are stated | Done |
| Claims are bounded by limitations | Done |

## Repository Structure

| Check | Status |
| --- | --- |
| Paper separated into `paper/` | Done |
| Experiment code separated into `experiments/` | Done |
| Figures separated into `figures/` | Done |
| Reports separated into `reports/` | Done |
| Raw pipeline evidence retained | Done |
| README gives artifact map and citation | Done |

## Style Checks

- No author email is included in the paper author block.
- The paper uses conservative research language.
- The paper avoids claiming validated employment prediction.
- The repository README presents results, limitations, and reproducibility
  clearly.
- Unicode em dash and en dash characters were checked and removed from authored
  Markdown and LaTeX files.
